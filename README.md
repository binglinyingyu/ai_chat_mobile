## 项目前言：
为了不方便使用电脑的情况下进行ai聊天，如外出或断电时编写。<br>
本项目只是为了方便在部分场景下的使用，所以相对其他ai聊天程式是很简化的，未来虽然会继续开发，但根本不准备作为其他程式的平替，只是为了让ai能够随时随地的陪伴我们，让没有电脑的用户也能够体验到无限制ai聊天的快乐。

## 功能特性

### 核心对话

- **会话管理** — 一个会话绑定一个角色，创建 / 重命名 / 切换 / 删除多个独立会话
- **重新生成 & 继续生成**
- **长按编辑** — 用户消息长按可修改，通过重新生成再处理

### 角色卡系统

- **SillyTavern V2 角色卡兼容** — 纯 JSON 格式，粘贴文本或选择 `.txt` / `.json` 文件导入
- **完整 CRUD** — 查看（易读卡片）/ 编辑（表单修改）/ 导出 / 删除 / 导入
- **会话内换角色** — 会话绑定角色可随时更换，不影响其他会话
- **用户卡** — 定义 AI 眼中的你，注入 system prompt
- **预置角色** — ⭐ 星野爱 · 🤖 DeepSeek（通用 AI）
- **方便修改** — 可以通过“通用ai”进行导出，然后再导入后便可在角色卡的卡片页面进行修改，不需要有改动json文件内容的经验！

### 智能增强

- **⏰ 时间感知** — 每条消息自动附加当前时分，AI 感知但不暴露时间戳
- **📋 全局提示词** — 三层架构（全局 → 角色 → 用户），可自定义所有角色共享的格式约定
- **🎨 文本分别渲染** — 对话语言金色高亮 `#FFD700`、心理描写斜体、代码块深色底
- **🔽 模型选择** — 下拉切换 `deepseek-v4-flash` / `deepseek-v4-pro`

### 数据管理

- **📤 会话导出 导入** — 会话级 `.txt` 导出（内容为 JSON）
- **📤 角色卡导出** — 一键导出角色卡为 `.txt`，方便分享和备份
- **🔒 隐私隔离** — 所有数据仅存浏览器 localStorage，分发 `chat.html` 不携带任何用户数据

### 视觉体验与交互细节
此处参考了fallenshadow主播的周表设计美学
- **移动端优先** — `viewport-fit: cover`、安全区域适配、键盘弹出平滑滚动
- **仅按钮发送** — 为了适配手机虚拟键盘，Enter 仅换行
- **生成中停止** — 发送按钮流式生成时变为停止按钮，可中途打断

## 快速开始

```bash
# 1. 克隆或下载本项目
git clone <repo-url>

# 2. 用浏览器打开（无需服务器、无需安装）
open chat/chat.html

# 3. 调取deepseek api
api调取链接（ds）：https://platform.deepseek.com/api_keys
> 为了方便,使用的deepseek api，暂未测试其他openai的情况
```

**首次使用：**

1. 点击右上角 ⚙️ → 填入 DeepSeek API Key
2. 点击「🔗 测试连接」确认通畅
3. 点击「保存设置」
4. 顶部会话 Tab 点击已有会话，或点击 `+` 新建
5. 输入消息，点击发送按钮开始对话

> **注意**：API Key 仅存储在浏览器 localStorage 中，不会上传到任何服务器。

## 📖 提示词编写指南

### 架构

整个系统的提示词分为三层，按优先级从上到下拼接为 API 请求的 `messages`：

```
┌────────────────────────────────────┐
│ 全局提示词（设置 → 全局提示词）     │  ← 所有角色共享
│   · 时间隐藏规则                   │
│   · 文本格式约定（引号/星号/括号） │
│   · 排版要求                       │
├────────────────────────────────────┤
│ 角色系统提示词（角色卡 JSON）      │  ← 每个角色独立
│   · system_prompt （核心指令）     │
│   · description / personality      │
│   · scenario / first_mes           │
│   · mes_example （对话示例）       │
├────────────────────────────────────┤
│ 用户 Persona （用户角色卡 JSON）   │  ← 定义 AI 眼中的你
│   · description / personality      │
└────────────────────────────────────┘
```

### 角色卡 JSON 格式
如果你习惯ai编写提示词，可以将本文件中任意角色卡导出后喂给ai，让其根据角色卡规范生成即可

```jsonc
{
  "spec": "chara_card_v2",
  "spec_version": "2.0",
  "data": {
    "name": "角色名",
    "description": "外貌与性格描述",
    "personality": "人格标签（一行）",
    "scenario": "开场场景描述",
    "first_mes": "新会话首条 AI 消息",
    "mes_example": "<START>\n{{user}}: 用户消息\n{{char}}: AI 回复\n<START>\n...",
    "system_prompt": "核心行为指令字符串",
    "post_history_instructions": "插入历史之后的指令",
    "creator_notes": "备注"
  }
}
```

### 关键字段说明

- **`system_prompt`** — 角色的核心行为指令，放在全局提示词之后
- **`mes_example`** — `<START>` 分隔不同轮次，`{{user}}` / `{{char}}` 为占位符
- **`first_mes`** — 仅在新会话 history 为空且用户发送首条消息时触发
- **`post_history_instructions`** — 插入在对话历史之后、当前消息之前，适合放每轮提醒的规则
- **全局提示词**中的规则对所有角色生效，不应在角色卡中重复

## 备注
- 为了适配部分手机无法在文件管理中显示json文件的情况，支持txt导入，同时也让导出为txt文件
- 部分手机可能在文件管理中不显示html，这个可以通过分享给聊天软件后在软件中选择使用浏览器打开
- 不同浏览器的存储读取情况没有测试，建议使用edge浏览器打开
- 目前无法更换头像，而默认会采用角色的第一个字作为头像嵌入字，可能很丑，见谅
- 如果有其他bug或影响使用体验的欢迎提出issue！
- 本项目的代码内容主要为ai辅助编写，可能会有部分错误，欢迎指出！
